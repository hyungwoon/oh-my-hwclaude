# 계약 이행 검증 (Contract Enforcement)

> 실행 중 에이전트가 플랜에서 이탈하는 것을 방지하는 런타임 검증 시스템.
> "A를 시키면 A'을 하고, 모든 다운스트림이 잘못되는" 캐스케이딩 실패를 차단한다.

> 프로젝트에 동일 목적의 로컬 규칙이 있으면 로컬 버전이 우선합니다.

## 핵심 원칙

- 에이전트는 플랜을 "합리적 근사치"로 대체하려는 경향이 있다
- A'은 A에 가까워 보이지만, A'에 의존하는 모든 코드가 잘못된다
- 일찍, 자주 검증해야 캐스케이딩 실패를 방지할 수 있다

## 검증 주기

| 모드 | 검증 주기 | 검증 에이전트 |
|------|-----------|-------------|
| 장시간 작업 | 매 3개 태스크 완료 후 | verifier (haiku, fresh context) |
| 병렬 작업 | 모든 병렬 태스크 완료 후 | verifier (sonnet, fresh context) |
| 단독 작업 | 매 유의미한 커밋 전 | verifier (haiku, fresh context) |

## 검증 절차

### 1단계: 검증 에이전트 스폰

검증 에이전트는 반드시 **fresh context**에서 실행한다 (실행 에이전트의 컨텍스트를 공유하지 않음).

입력:
- 원본 플랜 (`.omc/plans/` 또는 요구사항 문서에서)
- 완료된 태스크의 git diff (`git diff HEAD~N`)
- 플랜의 File Ownership 섹션

### 2단계: 3가지 검증 수행

**a. 의도 정합성 (Intent Alignment)**
- 구현이 플랜의 의도를 정확히 반영하는가?
- 플랜이 A를 요구했는데 A'을 구현하지 않았는가?
- 핵심 요구사항이 누락되지 않았는가?

**b. 스코프 준수 (Scope Compliance)**
- 플랜의 File Ownership에 없는 파일을 수정하지 않았는가?
- 플랜에 없는 기능을 추가하지 않았는가? (스코프 크리프)
- 불필요한 리팩토링을 하지 않았는가?

**c. 다운스트림 영향 (Downstream Impact)**
- 변경된 함수/타입의 호출자가 모두 업데이트되었는가?
- 인터페이스 변경이 모든 구현체에 반영되었는가?
- 테스트가 새 동작을 반영하는가?

### 3단계: 판정

| 판정 | 조건 | 행동 |
|------|------|------|
| **ALIGNED** | 3가지 검증 모두 통과 | 다음 태스크로 진행 |
| **MINOR_DRIFT** | 의도는 맞지만 스코프 약간 초과 | 경고 표시 후 계속 진행 |
| **DRIFTED** | 의도 불일치 또는 심각한 스코프 이탈 | **실행 즉시 중단** |

### DRIFTED 시 절차

1. 실행 즉시 중단
2. 이탈 내용 상세 보고:
   ```
   ⚠️ 플랜 이탈 감지

   원본 플랜: [플랜이 요구한 것]
   실제 구현: [에이전트가 한 것]
   이탈 유형: 의도 불일치 / 스코프 크리프 / 다운스트림 미반영
   영향 범위: [이미 이탈에 의존하는 코드 목록]

   권장 조치:
   - [ ] 이탈 코드 롤백
   - [ ] 플랜 수정 후 재실행
   - [ ] 이탈을 수용하고 플랜 업데이트
   ```
3. 사용자 결정 대기 (자동 진행 금지)

## 이탈 패턴 감지

자주 발생하는 이탈 패턴과 감지 방법:

| 패턴 | 설명 | 감지 방법 |
|------|------|-----------|
| **근사치 대체** | A 대신 "비슷한" A' 구현 | 플랜 의도 vs 구현 비교 |
| **스코프 크리프** | 요청하지 않은 개선/리팩토링 추가 | File Ownership 외 파일 변경 감지 |
| **조기 단순화** | 복잡한 부분을 스텁으로 대체 | 미구현 함수/TODO 감지 |
| **암묵적 결정** | 사용자에게 묻지 않고 설계 결정 | 플랜에 없는 아키텍처 패턴 도입 감지 |

## 통합 지점

- **`rules/autonomous-execution.md`**: 에러 복구 및 연속 작업 패턴과 보완적 관계
- **`rules/quality-gates.md`**: 편집 품질 게이트와 함께 최종 검증 수행
- **세션 종료 절차**: 세션 종료 전 최종 계약 이행 확인

## 금지 사항

- **절대 금지**: 검증 에이전트가 실행 에이전트와 동일 컨텍스트 공유
- **절대 금지**: DRIFTED 판정 후 사용자 확인 없이 자동 진행
- **절대 금지**: 검증 주기를 임의로 건너뛰기
- **절대 금지**: "거의 맞으니까 괜찮다"는 판단으로 이탈 무시
