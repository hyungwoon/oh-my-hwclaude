# 세션 후 엔트로피 정리 (Post-Session Entropy Cleanup)

> 세션 종료 시 레포지토리 엔트로피를 줄이는 자동 정리 시스템.
> "100번 반복하면 유지보수 불가능한 레포지토리가 되어 에이전트가 계속 혼란스러워진다."
>
> 프로젝트에 동일 목적의 로컬 규칙이 있으면 로컬 버전이 우선합니다.

## 배경

에이전트는 겨우 수용 가능한 수준의 코드를 작성하고 엔트로피 감소를 위해 아무것도 하지 않는다:
- 함수 X의 동작을 B로 변경했지만, 모든 문서는 여전히 동작 A를 참조
- 인터페이스를 변경했지만, 일부 호출자는 이전 인터페이스를 사용
- 새 유틸리티를 추가했지만, 기존 유사 유틸리티가 여전히 존재
- 주석은 3세션 전의 동작을 설명

## 실행 시점

**세션 종료 전 실행** — 기능 커밋 완료 후, 푸시 전 단계에서 수행한다.

## 정리 절차

### 1단계: 변경 분석 (Blast Radius Analysis)

```bash
# 이번 세션에서 변경된 파일 목록
git diff --name-only HEAD~{커밋수}

# 변경된 함수/타입 식별
git diff HEAD~{커밋수} -- '*.ts' '*.tsx' '*.js' '*.jsx'
```

### 2단계: 정리 에이전트 스폰

```
Agent(
  subagent_type="oh-my-claudecode:code-simplifier",
  model="sonnet",  // fresh context
  prompt="[정리 지시]"
)
```

정리 에이전트 입력:
- 변경된 파일 목록 + diff
- 프로젝트의 CLAUDE.md 경로

### 3단계: 6가지 정리 검사

**a. 스테일 주석 (Stale Comments)**
- 변경된 함수/클래스의 주석이 현재 동작과 일치하는가?
- JSDoc의 @param, @returns가 실제 시그니처와 맞는가?
- "이 함수는 X를 한다"의 X가 실제 동작인가?

**b. 미사용 임포트 (Unused Imports)**
- 변경으로 인해 더 이상 사용되지 않는 import가 있는가?
- 제거된 코드가 참조하던 모듈이 남아 있는가?

**c. 문서 정합성 (Documentation Consistency)**
- CLAUDE.md의 프로젝트 설명이 변경사항을 반영하는가?
- README.md(존재 시)가 업데이트되었는가?
- 인라인 문서(주석, JSDoc)가 현재 코드와 일치하는가?

**d. 호출자 정합성 (Caller Consistency)**
- 시그니처가 변경된 함수의 모든 호출자가 업데이트되었는가?
- 타입이 변경된 인터페이스의 모든 구현체가 반영되었는가?
- 이름이 변경된 export의 모든 import가 업데이트되었는가?

**e. 설정 유효성 (Config Validity)**
- 설정 파일(`.env.example`, `tsconfig.json`, `package.json` 등)이 코드 변경을 반영하는가?
- 삭제된 모듈/패키지가 의존성 목록에 남아 있지는 않은가?
- 새로 추가된 의존성이 설정에 제대로 등록되었는가?

**f. 중복 코드 (Duplicate Code)**
- 이번 세션에서 추가된 코드 중 기존 유틸리티와 중복되는 것이 있는가?
- 새 헬퍼 함수가 기존 헬퍼와 유사한 기능인가?

### 4단계: 자동 수정 vs 플래그

| 항목 | 행동 | 이유 |
|------|------|------|
| 미사용 임포트 | **자동 수정** | 안전, 부작용 없음 |
| 스테일 JSDoc | **자동 수정** | 시그니처와 맞추는 것은 안전 |
| 간단한 문서 업데이트 | **자동 수정** | 팩트 업데이트는 안전 |
| 호출자 시그니처 불일치 | **플래그** | 의도를 확인해야 함 |
| 중복 코드 | **플래그** | 통합 방법을 결정해야 함 |
| 설정 유효성 문제 | **플래그** | 삭제 vs 업데이트 결정 필요 |

### 5단계: 정리 커밋

자동 수정된 항목은 별도 커밋으로 저장:

```bash
git add [수정된 파일들]
git commit -m "chore: 세션 후 엔트로피 정리

- 미사용 임포트 제거
- 스테일 주석 업데이트
- 문서 동기화"
```

플래그된 항목은 사용자에게 보고:

```
엔트로피 정리 완료

자동 수정:
- [N]개 미사용 임포트 제거
- [N]개 스테일 주석 업데이트
- [N]개 문서 동기화

검토 필요:
- [파일:줄] — 호출자 시그니처 불일치: [설명]
- [파일] — 기존 유틸리티와 중복 가능: [설명]
```

## 정리 범위 제한

- 이번 세션에서 변경된 파일과 그 **직접 영향권**만 정리
- 프로젝트 전체 정리는 하지 않음 (스코프 폭발 방지)
- 영향권: 변경 파일을 import하는 파일 + 변경된 타입을 사용하는 파일

## 통합 지점

## 금지 사항

- **절대 금지**: 정리 없이 세션 종료
- **절대 금지**: 정리 범위를 전체 프로젝트로 확장 (이번 세션 변경분만)
- **절대 금지**: 플래그 항목을 자동 수정 (사용자 판단 필요)
- **절대 금지**: 정리 커밋과 기능 커밋을 합치기 (별도 커밋 필수)
