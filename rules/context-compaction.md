# 알고리즘 컨텍스트 압축 (Context Compaction)

> 세션 핸드오프 시 컨텍스트 충실도를 최대화하는 시스템.
> "컨텍스트 불안은 시간의 함수로 증가하며, 에이전트는 점점 더 세션을 끝내려고 절박해진다."
> 프로젝트에 동일 목적의 로컬 규칙이 있으면 로컬 버전이 우선합니다.

## 배경: 컨텍스트 불안 문제

- 프론티어 모델은 작은 태스크를 원샷으로 처리 가능
- 문제는 수백만 토큰을 소비하는 복잡한 멀티세션 작업
- 컨텍스트가 차면 에이전트는 코너를 자르기 시작
- Claude에서 특히 심각: 세션 후반부에 검증을 건너뛰고, 단순화하고, 빨리 끝내려 함

## 핵심 원칙

- 핸드오프 프롬프트는 **압축(compaction)**의 한 형태
- 프로바이더보다 사용자가 레포지토리 구조를 더 잘 안다 → 더 나은 압축 가능
- 충분한 디테일을 담되, 정보 밀도가 높은 방식으로

## 트리거 조건

| 조건 | 행동 |
|------|------|
| context-guard 75% 경고 | 핸드오프 프롬프트 자동 생성 시작 |
| 세션 종료 + 미완료 태스크 존재 | 핸드오프 프롬프트 필수 생성 |
| 사용자의 "나중에 이어서" 표현 | 핸드오프 프롬프트 생성 |
| `/compact` 실행 전 | PreCompact 훅에서 핸드오프 생성 |

## 핸드오프 프롬프트 생성 절차

### 1단계: 압축 에이전트 스폰

```
Agent(
  subagent_type="oh-my-claudecode:executor",
  model="haiku",  // 빠르고 저렴
  prompt="[핸드오프 프롬프트 생성 지시]"
)
```

압축 에이전트 입력:
- 현재 플랜 (`.omc/plans/`에서)
- 태스크 목록 + 완료 상태
- 최근 git log (무엇이 변경되었는지)
- 현재 작업 중인 파일 경로
- 마지막 오류/이슈 (있다면)

### 2단계: 핸드오프 프롬프트 구조

`.claude/handoff/handoff-{sessionId}.md`로 저장:

```markdown
# 세션 핸드오프: {작업명}
생성: {날짜시간}

## 의도 (Intent)
[우리가 달성하려는 것 — 1-2문장으로 핵심만]

## 진행 상황 (Progress)
### 완료됨
- [x] [태스크 1]: [1줄 요약 + 변경된 파일]
- [x] [태스크 2]: [1줄 요약 + 변경된 파일]

### 진행 중
- [ ] [태스크 3]: [현재 상태 — 어디까지 했고 무엇이 남았는지]

### 미착수
- [ ] [태스크 4]: [설명]
- [ ] [태스크 5]: [설명]

## 핵심 결정 (Decisions Made)
| 결정 | 이유 | 대안 (기각됨) |
|------|------|--------------|
| [결정 1] | [왜 이 결정을 했는지] | [고려했지만 기각한 대안] |

## 주의사항 (Gotchas)
- [다음 세션이 반드시 알아야 할 것 1]
- [실패한 접근법 — 다시 시도하지 말 것]
- [알려진 이슈/제약]

## 현재 상태 (Active State)
- 마지막 편집 파일: [경로]
- 실패 중인 테스트: [있다면 경로 + 실패 이유]
- 빌드 상태: [통과/실패]

## 다음 액션 (Next Action)
[정확히 다음에 해야 할 단 하나의 행동 — 구체적으로]
예: "src/api/users.ts의 validateInput 함수에서 email 필드 검증 로직 구현"
```

### 3단계: 충실도 체크

핸드오프 프롬프트 생성 후 자체 검증:
- [ ] 의도가 원본 요구사항과 일치하는가?
- [ ] 완료/미완료 태스크 분류가 정확한가?
- [ ] 핵심 결정의 "이유"가 명확한가?
- [ ] 다음 액션이 즉시 실행 가능한 수준으로 구체적인가?
- [ ] 주의사항에 실패한 접근법이 포함되어 있는가?

## 새 세션에서 핸드오프 소비

### session-start 연동

새 세션 시작 시:

1. `.claude/handoff/handoff-*.md` 파일 존재 확인
2. 가장 최신 핸드오프 파일 로드
3. 핸드오프 내용을 컨텍스트에 주입
4. 사용자에게 표시:
   ```
   이전 세션 핸드오프 감지됨
   작업: [작업명]
   다음 액션: [다음 액션 요약]
   이어서 진행할까요?
   ```
5. 사용자 확인 후 → 다음 액션부터 바로 실행
6. 핸드오프 파일은 소비 후 `.claude/handoff/archive/`로 이동

## 통합 지점

- **session-closing.md**: 미완료 태스크 시 핸드오프 생성 필수 (세션 종료 체크리스트에 추가)
- **autonomous-execution.md**: 장기 작업 중 75% 도달 시 자동 핸드오프 생성
- **quality-gates.md**: 핸드오프 충실도 체크 기준과 동일한 품질 게이트 적용
- **PreCompact 훅**: 압축 전 핸드오프 스냅샷 저장

## 금지 사항

- **절대 금지**: 핸드오프 없이 미완료 태스크를 남기고 세션 종료
- **절대 금지**: "다음 세션에서 알아서 하겠지"라는 태도
- **절대 금지**: 핸드오프에 모호한 다음 액션 ("계속 구현" → 구체적으로 뭘?)
- **절대 금지**: 실패한 접근법을 주의사항에서 누락 (같은 실수 반복 방지)
